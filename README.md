# 현대모비스 설치현황 관리 페이지

`설치현황_명세서.md` 기준으로 만든 웹앱입니다. 프레임워크 없이 순수 HTML/CSS/JS +
Supabase(토큰 검증 RPC 기반 조회/저장) + GitHub Pages 배포 구조입니다.

## 파일 구조

```
index.html            대시보드 + 사업장 상세(모달) 단일 페이지
css/style.css          전체 스타일
js/config.js           Supabase URL / anon key (직접 채워야 함)
js/app.js              데이터 로드, 권한 판별, 저장, 필터, CSV 내보내기

sql/05_add_network_fields.sql (과거 실행 이력용 — 이후 스크립트에 흡수되어 더 실행할 필요 없음)
sql/06_privacy_and_unit_locations.sql (과거 실행 이력용)
sql/07_contractors_fixed_two.sql (과거 실행 이력용 — 08번으로 대체됨)
sql/08_two_level_installers.sql (과거 실행 이력용)
sql/09_status_two_states.sql (과거 실행 이력용)
sql/10_add_mac_address.sql (과거 실행 이력용)
sql/11_assign_group_sites.sql (과거 실행 이력용)
sql/12_reset_installer_assignment.sql (과거 실행 이력용)
```

(실제 데이터 있으면) `04_real_site_data.sql` 순서로 실행하면 됩니다 (01에 최신 스키마가
반영되어 있음). 이미 이전 버전을 실행해둔 프로젝트라면 `13_site_group_assignment.sql`
하나만 추가로 실행하면 됩니다.

## 품목 정보

| 코드(내부) | 표시명 | 수량별 반복 입력 항목 |
|---|---|---|
| A | GST-502 | 없음 (설치위치 고정) |
| B | GX-8200 | 설치위치 — 실제수량만큼 반복 |
| C | GX-8200 TCP/IP | 설치위치 + IP/MAC주소/게이트웨이/서브넷마스크/호스트IP — 실제수량만큼 반복 |

품목 코드(A/B/C)는 DB 내부 식별자로만 남기고, 화면에는 표시명(모델명)으로 보여줍니다.

## 업체 구조 (2단계)

```
담당업체(총괄) — 링고벨 / 씨스콜CNS  (contractors 테이블)
  └ 설치업체(실제 시공) — 6곳       (installers 테이블, 담당업체 소속)
      └ 사업장(59개소)              (sites.installer_id 로 배정)
```

- **사업장 배정은 두 단계로 나눠서 할 수 있습니다** — 설치업체(A~F)가 아직 정해지지
  않았어도 담당업체(총괄)만 먼저 정해둘 수 있습니다.
  - `sites.installer_id`: 설치업체가 정해지면 채워짐. 이게 있으면 담당업체는
    그 설치업체의 소속으로 자동 결정된다.
  - `sites.group_id`: 설치업체가 아직 없을 때, 담당업체만 잠정적으로 표시하는 값.
    설치업체를 나중에 지정하면 그 설치업체의 소속으로 자동 덮어써진다.
  - 상세 화면에는 "담당업체"·"설치업체" 드롭다운이 각각 있고, 설치업체를 고르면
    담당업체 칸은 자동으로 맞춰지고 잠깁니다(직접 수정 불가). 설치업체를 다시
    미배정으로 돌리면 담당업체를 독립적으로 고를 수 있습니다.
- **배정은 마스터만** 사업장 상세 화면에서 할 수 있습니다.
- **권한**
  - 마스터: 전체 조회/수정 + 담당업체·설치업체 배정.
  - 담당업체(총괄, 링고벨/씨스콜CNS) 토큰: 자기 소속(설치업체 배정 여부와 무관하게
    담당업체가 자기 쪽인) 사업장을 전부 **조회 및 수정** 가능 (배정 변경은 불가).
  - 설치업체(6곳) 토큰: 자기가 배정받은 사업장만 조회/수정 가능.
  - 토큰 없음/잘못된 토큰: 아무 사업장도 보이지 않음.

설치업체 이름(현재 placeholder: 설치업체1~6)은 실제 업체명을 받으면 아래처럼 바꾸면
됩니다.

```sql
update installers set name = '실제업체명' where name = '설치업체1';
```

## 명세서 대비 보강/변경한 부분

1. **조회 자체를 토큰 기반으로 제한** — 명세서 원안은 "조회는 누구나 가능"이었지만,
   업체끼리 서로의 현황을 보지 못하게 해달라는 요청에 따라 바꿨습니다. `sites`/
   `installations`/`installation_units`/`contractors`/`installers`/`app_admins` 전부
   RLS를 켜고 직접 SELECT 정책은 하나도 두지 않았습니다 (anon key로 직접 REST 호출해도
   아무 것도 안 나옵니다). 대신 `get_sites`/`get_installations`/`get_installation_units`/
   `list_installers` RPC로만 조회하며, 토큰에 따라 결과가 자동으로 필터링됩니다.
2. **B(GX-8200)·C(GX-8200 TCP/IP) 설치위치를 수량별로 반복 입력** — 처음엔 품목당
   설치위치 1칸이었는데, 실제수량만큼 반복 입력하도록 바꿨습니다. B는 설치위치만,
   C는 설치위치 + IP/게이트웨이/서브넷마스크/호스트IP 를 `installation_units`
   테이블에 대수별로 저장합니다.
3. **담당업체(총괄) 2곳 + 설치업체(실무) 6곳, 2단계 구조** — 처음엔 업체가 1단계
   (사업장에 업체 하나만 배정)였는데, 링고벨/씨스콜CNS가 총괄하고 그 밑에 6개
   설치업체가 실제 시공하는 구조로 바꿨습니다. 사업장은 설치업체에 배정되고
   담당업체는 자동으로 유도됩니다. 배정은 마스터만 가능(`assign_installer` RPC),
   담당업체·설치업체 토큰 모두 조회 + 수정은 가능하되 배정 변경만 불가합니다.
4. **진행여부 2단계로 축소 + 담당업체별 통계** — "진행중" 상태를 없애고 미착수/완료
   2가지로만 구분합니다. 대시보드 상단에 전체 통계 카드 아래로 담당업체(링고벨/
   씨스콜CNS)별 담당 사업장 수·완료 수·완료율을 보여주는 작은 카드가 추가됐습니다.
5. **담당업체만 먼저, 설치업체는 나중에 배정 가능** — 설치업체 6곳 선정이 끝나기
   전에도 사업장별로 담당업체(총괄)만 미리 표시해둘 수 있도록 `sites.group_id`를
   추가했습니다. 설치업체가 정해지면 그 설치업체 소속으로 자동 동기화됩니다.
6. **사업장 참고 정보(주소/현장담당자/모니터링 PC) 추가** — "모비스 정보.pdf"에 있던
   사업장 주소, 현장 담당자명·전화번호·이메일, 모니터링 프로그램 설치 장소·PC IP를
   `sites` 테이블에 추가했습니다. 화면에서 직접 수정하는 항목은 아니고, 사업장 상세
   화면 제목 바로 아래에 참고용으로만 표시됩니다 (값이 없으면 그 줄은 안 보임).
7. **진행여부 다시 3단계로 (미착수/설치예정/완료)** — 2단계(미착수/완료)로 줄였던
   것을 "설치예정" 상태를 추가해 3단계로 확장했습니다. 설치 일정 달력 팝업에도
   상태별 색상 + 작은 텍스트 라벨로 반영됩니다. 같은 김에, 담당업체만 배정하고
   설치업체를 미배정으로 되돌렸을 때 이전 설치업체가 그대로 남아있던 버그도
   고쳤습니다 (`assign_group` RPC가 설치업체를 확실히 비우도록 수정).

## 배포 절차 (신규 프로젝트 기준)

1. **Supabase 프로젝트 생성** (supabase.com, 무료 티어) → Project Settings → API
   에서 **Project URL**, **anon public key** 확인.
2. Supabase SQL Editor에서 `sql/01_schema.sql` 전체 실행.
3. `sql/02_seed_placeholder.sql` 실행 → 59개 사업장 + 담당업체 2곳(링고벨/씨스콜CNS)
   + 설치업체 6곳(placeholder)이 생성됩니다. 실행 결과에 나오는 `access_token` 값을
   저장해두세요.
4. `sql/04_real_site_data.sql` 실행 → 사업장명과 A/B/C 품목별 예정수량이 실제 값으로
   갱신됩니다.
5. `js/config.js` 를 열어 `SUPABASE_URL`, `SUPABASE_ANON_KEY` 를 1번에서 확인한 값으로
   교체.
6. GitHub 저장소를 만들고 이 폴더 전체를 push.
7. 저장소 **Settings → Pages** 에서 배포 소스를 브랜치(예: main) / 루트로 설정 →
   공개 링크 생성 (`https://<계정>.github.io/<저장소>/`).
8. 배포된 링크에 담당업체·설치업체별 토큰과 마스터 토큰을 붙여서 배포합니다.
   - 담당업체(총괄): `https://.../?token=<contractors.access_token>`
   - 설치업체: `https://.../?token=<installers.access_token>`
   - 마스터: `https://.../?token=<app_admins.access_token>`
   - 토큰 없이 접속하면 아무 사업장도 보이지 않습니다 (의도된 동작).
9. 마스터 링크로 접속해서 59개 사업장 각각에 설치업체를 지정합니다.

Supabase SQL Editor에서 `sql/17_status_three_states_and_group_fix.sql` 하나만

수정 + 진행여부 3단계 확장). 기존에 입력해두신 사업장 데이터는 전혀 바뀌지 않습니다.

## 로컬에서 미리보기

`index.html`을 더블클릭해서 브라우저로 직접 열어도 됩니다 (`file://` 경로로도
Supabase 연결이 정상 동작합니다). 별도 서버가 꼭 필요하면:

```powershell
python -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

## 화면 동작 요약

- **대시보드**: 전체 통계 카드(전체 사업장/완료/완료율) + 담당업체별(링고벨/씨스콜CNS)
  미니 통계 카드(담당 사업장 수/완료 수/완료율 — 카드의 📅 아이콘을 누르면 그 담당업체
  소속 사업장들의 설치일을 월별 달력으로 보여주는 새 창이 열림), 조회 가능한(=자기
  권한 범위의) 사업장 표(담당업체-사업장-설치업체-상태 순, 담당자 칸은 없음),
  상태별 배지(미착수/설치예정/완료), 품목별(GST-502/GX-8200/GX-8200 TCP/IP) 예정/실제 수량,
  검색/상태/담당업체/설치업체 필터.
- **사업장 클릭 → 상세 모달**: 설치업체 배정(마스터 전용, 담당업체별로 묶어서 표시),
  완료여부/설치일/담당자명/비고, 품목별(A/B/C) 실제수량, B(GX-8200)의 수량별 설치위치는
  번호+칸 하나로 압축 표시, C(GX-8200 TCP/IP)는 수량별로 설치위치/IP/MAC주소/게이트웨이/
  서브넷마스크/호스트IP를 입력한 뒤 **맨 아래 저장 버튼 하나**로 한 번에 저장합니다.
  권한이 없으면 모든 입력이 읽기 전용으로 표시됩니다.
- **마스터 토큰 접속 시**: 전체 사업장 조회/수정 가능 + 사업장별 설치업체 배정/변경 +
  CSV 내보내기 버튼 노출 (CSV에 담당업체/설치업체, B/C 수량별 설치위치·네트워크
  정보도 포함됨).
